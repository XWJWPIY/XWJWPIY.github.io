前言

Github 與 Github Page 簡介

Github 是目前最常用的 Git 存取站點，是個能將程式碼雲端託管平台，支援許多程式語言的存取，像是 Python、C++ 等，此外，也有支援建立分支以及版本控制的工具，並允許開發者和團隊一同開發專案。

而 Github Page 則是擁有 Github 帳號時可以享有的功能之一，每支帳號(或團隊)可以透過自己的使用著名稱，透過自己的帳號依照指定格式命名專案，可以建立一個 <使用者名稱>.github.io 的網站位置，相當於透過 github 架設了一個屬於自己的網頁。

Hexo 簡介

由於我對於 Node.js 與 Hexo 是完全初學，因此關於 Hexo 的介紹是引用自網路上的解釋：

Hexo 是基於 Node.js 開發的套件，它可以將 Markdown 轉成 HTML。 Hexo 可以快速幫你建立個人部落格的工具，在 Hexo 官網中可以找到許多別人已經做好的模板，並套用在自己的網頁上。

架設 Hexo 步驟

在正式安裝 Hexo 之前，要先安裝 NPM 以及 Git，前兩者安裝完畢後才能安裝 Hexo

  • 安裝 Git : 到 Git 官網下載
  • 安裝 NPM : 到 Node 官網下載 NVM 安裝 NPM

如果已經裝好 Git 及 NPM ，可以直接跳到後面裝 Hexo

安裝 Git

裝 Git 是因為我想利用 GitHub Pages 來架設網站，而 Git 可以幫助我們將處理後的網站檔案同步到 GitHub 的伺服器，使網站可以在網路上公開。

將 Git 安裝到電腦環境方式很簡單，基本上一路按 Next 下去就可以，除非有需要特別的設定，不然安裝時使用預設就可以了。

安裝 NPM

因為 Hexo 是基於 Node.js 開發的套件，因此如果要使用 Hexo 的指令，就必須使用到 NPM 的指令協助。

而 NPM 全名為 Node Package Manager ，是一個線上套件庫，可以理解為一個線上套件庫，用來從線上下載各式各樣的 Javascript 套件。

我們可以透過 NVM 來安裝 Node.js 跟 NPM。

接下來進入安裝的環節
首先，到 Node 的官網下載安裝包，如果想要自己選版本的話可以點擊下方的 package managers 選版本
我是點進 package managers 下載 v22.13.0 版的 .msi 檔

接著直接啟動安裝包，我們只需要安裝 npm package manager，其餘的設定保持預設即可

最後開啟 CMD 並輸入 npm -v ，檢查一下是否安裝成功，如果有出現版本編號代表有安裝成功。(安裝成功的可以直接跳到下載 Hexo 的部分)

安裝成功會顯示版本如下


不過在使用 CMD 時有可能因為電腦的指令權限設定，在輸入 npm 相關的指令時會出現以下錯誤訊息

可以在 CMD 中輸入 Get-ExecutionPolicy 確認目前的權限狀態

如果顯示的是 Restricted ，代表你無法使用部分的指令功能

因此，我們需要改控制的權限，在 CMD 輸入 Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass 可以暫時開啟指令操作權限，生效範圍就是你當下的 CMD ，當你刪掉視窗就會回到一開始限制的設定。

之後再次到 CMD 中輸入 Get-ExecutionPolicy 確認目前的權限狀態
如果顯示的是 Bypass ，代表你成功開啟權限

此時再輸入一次 npm -v 確認是否有顯示版本，如果有顯示版本就代表安裝成功。

安裝 Hexo

在安裝之前請先確認自己的指令控制權限是否為 Bypass 狀態，可以 參考上方 確認自己是否已經開啟指令的權限， npm 的指令需要在 Bypass 的狀態下才可以使用。

接下來就要利用 CMD 的指令安裝 Hexo 路徑用預設的 C:\Users\<username> 就可以了，並輸入以下指令

npm install hexo-cli -g

接著就會自動安裝 hexo ，在安裝完後使用在 CMD 輸入 hexo v 指令確認是否安裝成功，有出現版本號代表安裝完成。

之後就可以用 hexo 指令創建一個新的部落格專案

hexo init <自訂的新專案資料夾名稱>

專案資料夾名稱可以隨便取，我是以 hexo-blog 作為專案的名稱，如果成功應該可以在對應的路徑中找到資料夾
接著就利用 cd <剛創的專案資料夾名稱> 進到資料夾裡
之後再輸入 npm install 載入套件

然後可以使用 hexo server 或 hexo s 啟動 LocalHost 的 Server。(LocalHost 指的是只能從本地瀏覽此網站，外網無法瀏覽)

在瀏覽器輸入 http://localhost:4000 ，或是對 CMD 中的該網址名稱按住 Ctrl 並點擊左鍵，就能點開該網址，在本地就看見即時的變更了。

設定 Github Page

除了電腦環境配置以外，我們還需要先準備好對外發布的地方。 Github Page 是 Github 提供的、能讓開發者利用 git 的形式，配置好靜態頁面的功能。

我們要先在 github 上創建一個新的 Repository ，而且 Repository name 一定要照以下的方式輸入。
(username 一定要填自己 github 帳號的名字)

之後就會生成一個新的 Repository，Github Page 會用這個 Repository 的內容外顯示。
剛剛輸入的 Repository 名稱 <username>.github.io 會是你個人 Github Page 的網址，可以直接透過瀏覽器輸入網址找到。
(應該會是空的，因為你還沒有加入東西在裡面)

更換主題

我這邊是使用許多人推薦的 NexT 主題當作部落格的主題樣式，安裝方式有兩種 :

  1. 可以直接到 next 下載並解壓縮放在 themes 資料夾底下，並改名為 Next
  2. 或是用 git 的方式將檔案下載到 themes 資料夾底下，並改名為 Next

除了預設的 landscape 與 next 外，官網也有提供很多其他創作者開發的模板，可以從 這裡 點進去

接著點開最外層的 _config.yml ，把 theme 後面的主題名稱修改成Next。

之後在 CMD 中輸入 hexo s 重新啟動本地伺服器，就能看到更改後的主題成功換到 blog 上了。

發文

在發文前，我們先到最外層的 _config.yml 中，將 post_asset_folder 的權限改為 true ，這樣未來發文時，會自動在 source/_posts/ 底下建一個與文章同名的資料夾，縮有該文章的圖片可以丟到該資料夾底下

接下來就可以新增貼文了！試著創建一篇叫做「第一篇發文」的文章

在 CMD 輸入 hexo new post 第一篇發文 後，會發現在 source/_posts/ 多出一個 .md 的檔案，你這一篇的所有貼文內容都是用 markdown 語法寫在這個 .md 檔裡面，如下方示範。

如果要發包含圖片文章，要先在 CMD 輸入 npm install https://github.com/CodeFalling/hexo-asset-image --save 安裝 hexo 處理圖檔的工具，這樣圖像才會正常的顯示在 server 上。

圖像記得拉到 source/_posts/<文章名> 底下，並將圖片檔名拖拉到 .md 的內容中即可 (注意只需要檔名即可，在資料處理時程式會自動找與文章對應的資料夾底下搜索)

然後在 CMD 依序輸入 hexo clean,hexo generate,hexo s 就能看到本地伺服器中的部落格呈現出圖片了喔

透過 Github Page 上架成品

初次上架

接下來就要把你的成品推上 github 了，我們要先在 CMD 安裝 Git 套件 npm add hexo-deployer-git

然後到最外層的 _config.yml ，找到 deploy 後修改內容。 Branch 的部分由於 github 是默認 main 分支來顯示，因此我們先直接推到 main 分支

接著在 CMD 輸入 hexo d 或 hexo deploy ，就可以把文章放到 Github Pages 了
過幾分鐘後輸入 <你的 github 名稱.github.io> 就能看到上架的部落格囉～

未來更新

未來有文章要上傳，在文章打完後，在 CMD 輸入 hexo clean,hexo generate, hexo d 才能有效更新喔～
